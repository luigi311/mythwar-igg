1. Introduction to the Chatting System

About the Chat System

 The purpose of the chat system is to bring the players closer together and to provide a means of communication. You can type in your messages at the bottom left corner of the game screen. Upon pressing enter, your message will be displayed in the chat window. The button at the bottom left corner allows you to toggle between the messages outside of the game screen or inside the game screen. You can choose the channel to talk in based on the audience you want to reach out .to. Currently we have a few different modes : Local, Party, PM, Guild, Market and World. Emoticons are also enabled in the chat system. To use them, just key in the ??character and follow by the number from 0 to 99

About Chat Interface

  • The chat interface comprises of the following : chat display, function keys, input display and channel list display.
  • Chat display : to display the message posted by players.
  • Messages are displayed in this format : [Channel][player¡¯s nick][message content]
  • When the game is in full screen mode, the chat display can only be on the screen.
  • When the game is in window mode, the chat display can either be on the game screen or in another window.
  • When the game is in full screen mode, you can only choose to have the chat display either on the game screen or choose to turn off the chat display.
  • The system will be able to auto adjust the size of the chat display based on the following:
  • When the monitor is set to 800 x 600 pixel, and the chat display is set to display in another window, the player will not be able to see the game screen.
  • When the monitor is set to 1024 x 768 pixel, and the chat display is set to another window display, the game screen will be changed to top left hand corner at co-ordinates (0,0). The chat display can be dragged to any of the 4 side of the game screen. When the chat window is at either the left or right of the game screen, the chat display will be fixed at 200x600 pixels. When the chat display window is either at the top or bottom, the chat display will be at 800 x 138 pixels.
  • When the monitor is set at 1280 x 960 pixels, and the chat display is set in another display window, the chat display by default is at the side of game window (it can be drag to other 4 sides of the game screen), the chat window can be adjusted between 240x600 pixels to 480 x 600 pixels. When the chat display is at the top or bottom of the game window, the size of the chat display window can be set from 800x165 to 800 x 330.
  • The system will set the game display window to be the parent window and the chat display window to be in the child window. Hence the chat window does not have any control button. When the game window is being closed, the chat window will be closed too.
  • Chat input method: when there¡¯s no other input window being open, the message box at the bottom left hand of the game window will be the input window by default. Players just have to click there and type in their messages and press [Enter].
  • Chat function keys:
    - Display switch: toggle between in screen display or chat window display.
    - Scroll up: to scroll the message up
    - Scroll down: to scroll the message down
    - Chat history: Display the history for up to 1024 messages.
    - Emoticons: Display the emoticons available to choose
    - Channel: Select the channel to display your message.
    - Maximize the chat window
    - Message window
    - History: the system is able to store up to 16 lines of message.

2. Introduction to Channels

Channels

 By default, the font colour is set to white. The nickname of the player will precede the message. The channel colour will be in a different colour.

Local Channel

  • The chat dialog box will appear at the head of each character.
  • All players on the same screen will be able to view the message.
  • There are no level limits to using the chat system.
  • The messages will stay on the screen for 3 seconds.

Party Channel

  • The chat dialog box will appear at the head of each character.
  • All players belonging to the same party will be able to view the message.
  • There are no level limits to using chat system.
  • The message will stay on the screen for 2 seconds.

Guild Channel

  • The chat dialog box will not appear at the head of each character.
  • All players of the same guild will be able to view the message.
  • Only players in a guild will be able to use this function.
  • The message will stay on the screen for 3 seconds.

Market Channel

  • The chat dialog box will not appear at the head of each character.
  • All players at the same screen will be able to view the message.
  • There are no steadfast rules to using this channel but messages should adhere to the following syntax : ¡®sell? ¡®buy? ¡®purchase? ¡®offer?and ¡®request?along with the name of one of the items.
  • When the market channel opens, right click the items you want to sell off, the system will auto input the item name. When other players see it, they¡¯ll be able to click on the item and get more information on the item.
  • The messages will stay on screen for 30 seconds.
  • In the event that the GM finds out any abuse of the system by players, warnings will be given out.

World Channel

  • The chat dialog will appear at the head of each character.
  • All players on the same screen will be able to view the message.
  • There are no level limits to using chat system.

Private Channel

  • The chat dialog will not appear at the head of each character.
  • All players at the same sever will be able to use the system.
  • There are no level limits to using the chat system.
  • Right click on the player to chat with and he¡¯ll be able to see your message.
  • The message will stay on the screen for 2 seconds.

System Channel

  • System messages sent out by the GM will be in the colour RED.

Appendix

  • The chat dialog above the character¡¯s head will not be able to follow the character and will disappear after 5 seconds.
  • The maximum number of chat dialogs that appear above the character¡¯s head is 3.

Function Outline

  • Works in a similar way to MSN Messenger. The main purpose is for players to communicate amongst each other.

Sending messages

  • Messages can be classified into different channels.

  • Sending message interface : recipient nick, add as friend, delete, block and PM
  • Receiving message interface : sender's nick, add as friend, delete, block and PM

  • Players can send mass messages to other players.
  • The maximum number of characters is 256 words.
  • A maximum of 250 offline messages can be stored.

Block

 This function is for players who do not like to receive message from certain players.

  • The system will ignore all messages sent to the players by the particular player.

Online Player Search

  • You can search for players using their nick or ID.

Message History and Recording

  • There are 2 types of history: chat history and message history.
  • Chat history can store up to 1024 lines.

Team Play

 

1a. Teamplay

Forming a team is one of the ways for adventure and gameplay in Myth World. Players who are in a team will have a lighter burden in defeating the monsters; and some quests require a team to accomplish.

1b. How to form a team

Form your own team by making yourself a leader. To do that, you can click on the Flag icon (Alt+T), and your cursor will turn into a flag. Move the flag cursor to yourself, and there, you are a leader of your own team.

1c. Team leader's rights

  • The team players have to follow the leader wherever he/she goes.
  • The team leader has the right to approve or disapprove a player¡¯s request to join the team. He/she has to click the ¡°Confirm?or ¡°Delete?button, according his/her decision.
  • The maximum players you can have in a team is 5.
  • The team can disband a team if he/she has decided not to lead the team anymore. Click on the ¡°Leave Party?button in the Party window to disband the team. The flag icon will no longer be on the player. However, disband can happen under possible circumstances like: server down, death, or team members withdrawing themselves.
  • The leader also has the right to kick a member out of the team. Click on the member you wish to remove, and then click on the ¡°Kick?button in the Party window to remove him/her.

2. Joining a team

If you do not have team, you can join a team which is already formed. Look for the team leader and move the flag cursor over to him/her, and wait for an approval.As a team member, you are automatically set to follow your leader wherever he/she goes. You are entitled the right to leave the party. To do that, launch the party window and click on ¡°Leave Party?button to withdraw yourself from the team.

3. Team features

  • The team can only have 4 members and 1 member. The least is 1 member in a team.
  • The team can have private conversation among themselves, away from the world channel. Just check on the ¡°Party?option from the channels. The conversations will be presented in green color.
  • You can find out your fellow members?details by clicking on the Party icon (Alt+T). The party list will appear and arrangement is done according to the player¡¯s level; the leader will always appear on top. Click on the ¡°Display List?button to reveal their basic information.
  • The basic information includes the in-game name, level, gender, race, race title, and guild and group emblem.

4. Guild Items

  • Guild items are special supplements that have unique attributes added to the user and his/her affected team member. The items are used to increase defense against physical and magical attacks.
  • Only the leader may wear the items, which could increase the whole team¡¯s fighting force and its value.
  • These guild items are placed with a very high value and help especially in the country war. For instance, the country leader wears the ¡°Kingdom¡¯s Crown? the overall country members and warriors will have a great increase in their attributes, making them a much stronger team against the opponents.
  • Group items are only effective when there are more than 2 members in a team.
  • In a team, only the leader can be equipped with these items and the selection of a group leader is very important, as the increment of attributes is dependent on the player¡¯s characteristics.

Trading

 

Trading System

1. Player to Player Trade and Give
Trade is one of the very important interactions in the game ?the trading behavior between 2 or more players.

Our team has been working hard to create a convenient trading system for the players; the lifespan and value of Myth War is heavily dependent on the trading system.

2. Trading Reality
Player to player trading (with no third-party or external player involved) is the most commonly-seen style in online games.

Trading Step 1: The player must click on the ¡°Trade?button and move his cursor to his desired trader to initiate a trade.

Trading Step 2: Upon agreeing on the trading terms, both parties have to confirm their offer and then they can proceed to click on the ¡°Confirm?button.

Note : 8% of the agreed transaction (which involves monetary) has to be deducted as a administration fee.

Giving System
The giving system in Myth War is such that an item or more is transferred to another online player.

There are 2 ways in which the players can give an item:
a. Giving to another player with your vicinity.
b. Distant Giving ?items can be transferred to your online buddies.

Giving Steps : Click on the ¡°Give?button and move your cursor to the desired player. Select your item and drop it in the giving field and click ¡°Confirm?button to proceed.

Note: The giving method is also applied to the quests that may need the player to give the quest items to the NPCs.

However, giving of items will not be successful if the receivers do not have enough space in their inventories. One adding advantage to the giving is that the relationship between the 2 players will increase and releasing the gift¡¯s special abilities.

PK system

1. The reason of having a PK system in Myth War

In the world of Myth War Online, there is bound to have clashes/disputes between players akin to the real world. There are many ways to resolve problems in the gaming or real world, but there is a special way in the gaming world, that is: Force PK (Player Killing), our objective is to let players know how it feels like to have "I-just-want-to-do-it-my-way" style thus PK-ing in MMORPGs is a must! Due to such situations, we call these types of people PKer (mass player killing player). There are laws in the gaming world as well, in order not to have chaos. As we speak, we are tweaking the rules for fair game play.

2. PK Rules

The 'Devil's Blood' which can be purchased from the grocery shop allows players to go into PK mode.

  • Once level 15 and above, players will automatically have the right to PK.
  • For a party PK situation, all the players have to consume 'Devil's Blood' before the battle begins; players that did not consume any will be automatically kicked out of the battle, thus unable to take part in the battle.
  • During a party PK battle, only the leader is able to initiate the battle with other players/party. Upon using 'Devil's Blood' and closing the 'allow challenge' button, players will be able to start PK-ing people. At this point of the time, player's mouse icon will change into the [Attack] button (alt + a) once the icon is clicked on any player, it automatically goes into battle mode, no matter what the outcome of the battle is, player's rating of good or evil will drop or increase by 10.
  • PK-ing players with red names will not drop rating even when battle is lost, or in some situations when the battle is won, rating will be increased instead of dropped. Players who consumed 'Devil's Tear' once engaged in a battle are not allowed to escape, but otherwise for players that are being PKed; they are allowed to escape.
  • Force PK cannot be carried out in town or places that do not have random encounters.
  • Once a leader initiates a battle with another party/player, all members in the party will take part in the battle.
  • Players are not allowed to PK guild members, his/her partner (married couples), and players who are in brotherhood, if a PK is initiated on an acquaintance the friendship level will be reset back to 0.
  • PK status affects the evil/good rating, reputation in game (color of the names) and the time a player will be jailed and the punishment upon death.
  • PK abuse protection system: In order to avoid players to abuse the PK system, 3 icons will be issued once a PK battle is over, normal PK icon - Killed in PK battle icon - duel icon, once accumulation of 10 counts of either 1 or mixture of any of these 3 icons is reached, no one is allowed to Force PK him/her and he/she is unable to Force PK others as well.
  • In Myth War Online, PK is only legitimate in death zones, thus having PK in the zone will not cause evil/good rating to drop and no PK icons will be issued.

3. PK Situations

  • In Myth War Online, a note will be dropped upon death of players in a PK match.
  • In Myth War Online, players killed in a PK match will get a notice saying who PK-ed him/her.
  • In Myth War Online, players who are killed during a PK match can set a curse on the PK-er. The chance of success depends on the level differences between the 2 players and relationship.

Duel System

1. Duel Style 1 - Battle Arena

  • Battle Arena: City - Island of athletics. There are two PK sports arenas, 1 for individual, 1 for groups/parties.
  • Individual Duel: In individual Duel mode, 11 areas are provided and 11 areas separately correspond to 11 ranking stages. After paying a certain amount of gold as a bet, it will be dispatched automatically by the system according to the player's level into different areas. Players are not allowed to form any party at any point or time within the arena and players are to search for their opponents within the arena. The winner gets to keep 80% of the bet and is allowed to stay on for more battles. Players who lose will be kicked out of the arena.
  • Party Duel: There are 4 arenas provided for party duels. These four arenas separately correspond to four kinds of team scale. The system will then automatically send the team into different arenas, depending on the Team Scale. Once in the arena, disbandment of the party will result in a disqualification and thus the whole team will be kicked out of the arena. The party leader will be the one looking for duels; Winners get to keep 80% of bets and losers get kicked out of the arena.

2. Duel Styles 2

  • Free mode: Player's condition will go back to normal after a battle and the winner gets a certain amount of gold.
  • Death match: Players level 15 and above will receive death as punishment when killed, but no ratings for evil/good will be given for both parties. The winner gets a certain amount of gold.

3. Differences between DUEL and PK

  • Those who die during a PK battle will receive death as punishment. Except for death matches, HP/MP of players will be restored back to their original state before the battle begins.
  • PK can be only carried out in areas for PK battles, whereas DUELs can be carried out anywhere, even in town.
  • PK will result in drops or increases of ratings for good or evil, whereas duels will always result in a gain in war experience. Betting is also allowed in Duels.
  • PK-ing is not allowed between guild members, partners (married couples) and Brotherhood alliances. PK-ing an acquaintance will result in a reset of friendship level , whereas dueling an acquaintance increases the friendship level.

Operation Guide

Getting Around
There are various ways to move your character around the game. Left clicking on a location on the screen and your character will walk there.
Running About
To make your character run to a location on screen, right click on the location.
Attacking
During a battle, left click on a monster to attack. You may also use the shortcut key Alt + A.
Magic Attack
You may also perform magic attacks on enemies. To do this, click the skill button and select a skill from the list that pops up. Move your cursor over the enemy/target and left click on the target, the attack sequence will then commence. Right click to cancel the magic attack sequence.
Drop
To discard an item, open up the inventory, select the item you wish to drop and then click the "drop" button. A prompt will popup to confirm your decision. Note: Once an item is dropped, it cannot be picked up, it is lost forever.
Chat
To chat, key into your message into the chat box and hit enter to send. You may select a few modes of chatting depending on who you wish to send your messages to.
Conversing with the NPCs
To converse with NPCs, right click on them to begin talking. You may then talk to them to retrieve information regarding quests, questions or trading.
Check Item
To display information about an item, move your cursor above the item for 2 seconds and information about the item will appear.
Using an item
To use an item, open your inventory, select the item you want to use then click on the "use" button.
Using Armor
Choose the armor you desire, and click the "Use" button. If a current item is in use, the system will automatically replace it with your selected item.
Removing an item
Click on the item you want to remove, then left-click.
Trade
Get near to the player whom you would like to trade with and right-click on the target. A window will appear that allows you to select the items in your inventory. You may also use the shortcut key Alt + X.
Give
Click the "Give" button, right-click the player you would like to give to. A window will appear that allows you to select the items in your inventory. The shortcut key is Alt + G.
Party
In order to form/join a party, a leader must first exist. If there¡¯s no current leader, you can click on the "Party" button to yourself to become a leader for the team. To join a party, click the "Party" button and select the leader for approval. Shortcut key is Alt + T.
Making Friends
Click on the "Buddy" button and drag the icon to the character you wish to make friends with.
Dismissing a pet
Click on the pet icon on the top left-hand corner of your screen to activate the pet window. Then select a pet from the list and click on the "Dismiss" button.
Replenishing MP/HP
To quickly replenish your health or manna, open up your inventory and select a fruit/potion for healing. Click the "Use" button.
 
How to battle

1. Fight system
The main characteristic of Myth War lies in the characters and the main theme is the country system. The country system involves country war. It is the main focus to start a war and it has a direct influence whether the system is a success or failure.
The battle system is one way players can level up, get special headband title. Disputes between players are usually settled through battle system.  
Thus the battle system is the core basic of Myth War, in order to let players enjoy a better battle system, we have made new changes to it.
The specialty of Myth War's battle system is flexible tactics, control and communication ,they are the points that made Myth War far different from other MMORPGs.

2. Attack Time Battle
The ATB (Attack Time Battle) system
Encounters system: Normal battle system is based on random encounter system, whereas quests, NCP or BOSS battles are only triggered by talking to NPCs.
Random encounter means you will not see any enemies/monster in the area of the maps.
The only way to engage in a battle is to be at the specific training ground like dungeon walking around the dungeon.
Walking for about 12 seconds to encounter enemy, the timing decreases progressively by 2 seconds if no battles are engaged after the 12 seconds after the previous battle.
Running for about 8 seconds to encounter enemy, the timing decreases progressively by 1 second if no battles are engaged after the 8 seconds after the previous battle.
If player's level is 3 levels lower than the training area, the decreasing timing will be +2seconds which is (4 seconds for walking , 3 seconds for running).
If player's level is 10 levels higher than the training area, the decreasing timing will be +5seconds which is (7 seconds for walking , 6 seconds for running).
The percentage of encountering monster is 60% and 40% without encountering one.
Other encounters with monsters are through quest given by NPCs. To complete the quest, players will have to challenge the NPC.
Duel system
To duel players or what we call the PVP (players vs players system), just click on the [Attack] button and click on the player you wish to duel with in order to start the duel.
If players are in PK mode (consume a 'devils blood' which can be bought from the grocery shop) and when the conditions are right, upon pressing the [attack] button, player will be able to enter a direct battle once the [attack] button is clicked.
For Guild/server wars/battles, once either side of the party leader activates the battle by clicking on the [Attack] button, players will enter battle immediately.
For Quests, NPC or BOSS in training areas/maps, players will just have to click on the [attack] button and select it on the NPC to activate battle.
Once players touch the monster walking on the map/dungeon, the battle begins.

When moving in a training area/dungeons, there will be certain timing and certain percentage for random encounters.
Maximum number of players for in-game battle:
The maximum number of players in a party is 5, each person is only allowed to bring in 1 pet for the battle, additional pets or hired mercenaries will be shown as a magical icon.
Battle rules: Half Round elimination rule, every battle last for 75 mins, once the timing hits 0, the battle ends. The movement in battle will be decided by the ATB bar, if the charge speed of the ATB bar for both sides are the same, there will be a 20% chances of hitting each other together, statistics of the pets will be back to the original point unless they died during battle.

Movements
Every character's movement is based on their own agility statistics points.
During the process of the charging of the ATB bar, commands can be set.
Once the ATB bar is charged, the command will be carried out, during that process, all the other players' ATB bar will be frozen.
Once the ATB bar is charged, if no commands were given by the player, an auto attack on a random target will be done.
Once the command is being executed and player reaches back to his/her original position, the other players' ATB bar will start to recharge again.
You may call for a stop during battle, each time for 20 seconds and uses up MP of players.
Winning conditions: To win a battle, either side must be unable to fight or escape from battle. The percentage of escaping from a battle is 80%, so think wisely before you want to escape or else that might be the last turn for your character. You cannot escape during boss battles or quests. There is a special condition during PK, where both players are down and left with their pets in battle, whichever pet wins the battle is considered the winner and will be restored back to original HP/MP.
You will be rewarded with Gold/Experience/Items for winning battles for Quests and NPCs. It will be divided equally around the party if the Quests and NPCs battles are done in a party.
No experience or gold will be gained during PK, but war experience can be gained after the battle and the player will be judged as either good or evil. If it is a group PK the war experience will be counted by the last hit done by the last person, thus same goes for the good and evil.
You will be rewarded with Gold/Experience as well for taking part in Guild battles and Server Battles. Experience or gold will be divided equally in Parties/Guilds immediately after the battle.
Death during battles: when the battle ends, if the player still remains in a state of unable to battle, he/she will be pronounced as dead, thus player will be removed from the party, depending on different death situations different outcomes will occur, please refer to the respawning section for more info.
Status : the status system used is overlapping system; there cannot be more than one status in a battle during a one time casting, on one enemy or player, for more information please visit the characters chart system.

Other rules
Players are not allowed to change pets during battles.
When using the defensive option during battle, the damage reduced will be 50% lesser.
During Quests/BOSS battles, the same enemies will only spawn maximum up to 3, this is to increase the interactive level of the game.
Pet's loyalty: The movement of the pets lies on the percentage loyalty of the pet, if the pet's loyalty is less or equal to 10%, there will be a 90% chance that it might escape, 10% chance of attacking the owner, if the loyalty is less or equal to 30%, there will be a 70% chance that it might escape, 30% chance that it might attack the owner. Pets having less or equal to 50% loyalty will have 50% chance of attacking players themselves, or 50% chance of executing the command of the player. Pets that are less or equal to 60% loyalty will have 40% chance of random attack on enemy target, 60% of player's command, pet's with more than 60% loyalty will have 80% chance of executing player's command, 20% of random attack on random enemy.
Capturing of pets: Pets will appear in the five main cities of the pet farm, wild or rare pets DO appear in other places other than the pet farm (under the control of GM). Encountering will be same as NPC encounters, when the player gets in contact with the pet, a battle begins. When in battle, only one of the monsters you see is a pet. Out of the three you are fighting, the rest are normal monsters. If player is unable to capture pet during battle, the pet's location will automatically be reset to other places of the city, if pet is killed during battle, it will disappear.

Battle Controls
1. Positions
a. Player's positioning
On a 800x600 resolution screen, during a battle, the positioning of twenty players is fixed even though the background may not be the same.
Once after a command is being carried out, players will be positioned back to the original position before the command is carried out.

b. Sequence of the position
The leader will always be in the middle of the line, followed by 2 players by each side (left and right)
Players who are not in a party will always be in the middle during battles.
Pets will always be in front of player.
Player's party will appear in the lower left of the screen, while the enemy will appear on the upper right.
If enemies are Monsters/NPCs, the positioning will start from the back row onwards, towards the front but still in two straight rows.

c. Three types of encounters
Sneak attack: Enemies in battle will not be able to escape and requires a turn to move back to the original status.
Ambush: When being ambushed, player's party and pet will not be able to escape with backs facing the enemy, thus wasting a turn to return to the original state.
Head to head: Everything remains the same during this encounter with no handicap on either side.
During random encounters, any of the three situations may appear on random basis.
When player initiate to approach the enemy, it will be counted as normal encounter, but if enemy approaches the player, it will be counted as ambush/sneak attack.

Battle statics
a. Types of attack
Normal attacks: most commonly used attack, using weapon to attack the target.
Skill: uses the skill in the battle, each skill uses MP, but does a lot of damage than normal physcial attacks.
Combo: Depending on the statistics of the player, items of the player as well, the system will calculate the chances of player doing a combo attack. The damage of the combo attacks are being reduced by half 1 hit after another.
Combined attacks: during a party, if it happens that both players' ATB bar are fully charged at the same time, there will be a 20% chance a combination attack will be executed, causing more damage.
Berserk: Depending on the level/statistics/item of the player, a berserk attack will be done, berserk meaning damage of player will increase to a certain percentage, same goes for defense and players ignore enemy's defense as well.
Critical: Depending on the level/statistics/item of the player, there will be a critical attack, it is an attack that is 50% more damage than a normal attack, but does not ignore the enemy's defense.
Items: Items that are able to attack enemy/target. Most commonly used is the seal stone for capturing of pets.
Commanding your pet to attack: The system will announce your pet's attack when it is its turn.
Mercenary attack: The system will announce your mercenary attack when it is his/her turn.  
Strategic Pet Attack: The player can strategize to allow his/her pet to duel. However, the success rate is subjected to the loyalty level of the pet.

b. Attacks
Attacked: Attack against the player can be carried out when the damaged reached 5% of the current HP.
Knockout: The player will be defeated and unconditioned to enter into the battle again when the damage against him/her is too great.
Dodge: Avoiding attacks from opponent.
Health & Mana Leech: Increase own health and mana through attacking your opponent.
Acknowledgement: The player can acknowledge a monster as a pet, turning it into an egg, then baptize it to become your pet.
Block: up to 70% chance of blocking.
Defense: Can decrease 50% of damage rate.
Retaliate (Physical): The attack rating is determined by the damaged you have received from the opponent.
Retaliate (Lightning): The attack rating is determined by the damaged you have received from the opponent.
Retaliate (Arrow): The attack rating is determined by the damaged you have received from the opponent.

Other Behaviors
Couple attack combo: When there is a couple in a team, or when the couple acquired the combo skills ofthe system will distribute special skill attacks to increase the team attack rating, at the same time heighten the friends' relationship.
Buddy attack combo: When there's a buddy in a team - the system will distribute special skill attacks to increase the team attack rating, at the same time heighten the friends' relationship.
Dismiss: Dismiss a pet from a battle, at the expense of deducting its loyalty point to 1.
Escape: Escapes during a battle. But if you are in a battle, escaping also includes leaving the party.
