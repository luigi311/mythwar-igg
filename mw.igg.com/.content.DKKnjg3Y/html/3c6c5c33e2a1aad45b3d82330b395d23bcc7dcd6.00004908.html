[Security] What's my computer doing!?

Some people are already aware that Windows comes bundled with a CLI tool called "netstat", for those who aren't, this is an introduction of what it is, what it does and how you can use it to keep your computer safe (ish).

No amount of security will protect you, partly because security companies can't do fore-sight, though they do attempt to block commonly used holes and scripted commands etc...

Anyway,

What is netstat
Netstat is a tool which you can use to find out various bits of information about the ports, destinations and programs using the internet on your computer.

While it doesn't show utilisation of each item, it should help you identify any suspicious programs and such running and you can take action on them.

*[text] denotes a key press

How do I use it?
Open a Command Prompt window.

Start &gt; Run
[windows key] + [r]

At the new prompt - type: cmd [enter]

We should be presented with a lovely, traditional Command Prompt window - looks like this



To display a list of currently active connections, just type in

netstat [enter]



So for me, this displays three active connections - I don't run a lot.

If you wanted to know, They are Google Talk & Windows Live Messenger.

This isn't really handy info since I already knew they were running, but for some viruses, trojans and malicious ware, they manage to run under much more "hidden" means...

so, netstat comes fully equipped with a bunch of various switches which turn on various other features!

-a | This displays all connections and all listening ports.
-b | Displays the executables which own each connection - if you combine this with -a, you can find out all the DLL files which windows is using to deal with all this.
-e | Display "bytes recieved/sent" info
-n | Displays address as IP's
-o | Displays the process ID which owns the connection
-v | Used with -b, this will display the sequence of components used in creating or listening to the port for each executable.

xxx | Adding any number to the end asks the netstat command to run every 'xxx' seconds as you specified, to cancel use [ctrl] + [c]

Interpreting all the info
My personal opinion is to use netstat -abv 600

This gives us everything that's going on, it tells us the executable and it also tells us all the DLL files involved (usually) and refreshes every 10 minutes - you don't have to use the time if you don't want to.



Be-aware that any command Prompt window does have a line limitation on it, I can't remember what it is though, I think it's 1024 lines... probably is less actually but anyway...

Interpreting what's going on is fairly straightforward. If you come across anything which looks and or sounds suspicious, for example, an executable called run.exe (as i found earlier), simply whack it into a google and usually a whole host of websites will return results related to that executable and tell you what it is...

The connection state's I found are explained below:

Established connection is active and open for use, data is usually being transmitted or received.
Listening Connection is open for connection.
CLOSE_WAIT Not fully sure, but seems to be when Client closes, but the server does not acknowledge the initial close request, so the client is left waiting for the handshake to close.
TIME_WAIT Connection has been or is being closed, TIME_WAIT can remain for anywhere up to 4 minutes to be sure that data is sent and recieved - as is the way TCP works.

There are a few other states which involve the creation of sessions - SYN, which becomes Established when fully connected and FIN when attempting to negotiate application termination or session ending.
